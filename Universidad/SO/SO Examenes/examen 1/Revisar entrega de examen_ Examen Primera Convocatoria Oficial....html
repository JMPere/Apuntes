          Un microcontrolador dedicado exclusivamente a la gestión de dispositivos gestiona mediante entrada/salida guiada por sondeo tres dispositivos DSP1, DSP2 y DSP3, mediante el siguiente bucle de control en el que el dispositivo DSP1 se atiende por prioridad escalonada respecto a DSP2, y DSP3 se atiende por prioridad uniforme respecto a DSP1 y DSP2:

          while(true)
          {
            if (dispositivo_preparado(DISP1))
              atender_dispostivo(DISP1));
            else
              if dispostivio_preparado(DISP2))
                atender_dispositivo(DISP2));

            if (dispositivo_preparado(DISP3))
              atender_dispositivo(DISP3));
          }

          A la vista del bucle de control, y sabiendo que los tres dispositivos tienen el mismo tiempo de servicio T, ¿A qué dispositivo se le puede garantizar un menor tiempo de respuesta al dispositivo? 
          Respuesta seleccionada:
          [No se ha dado ninguna]
          Respuestas:
          Comentarios para respuesta:

          Pues no. Veamos para cada dispositivo qué se puede garantizar:

          DISP1:  Este dispositivo, tras quedar preparado para recibir servicio, el peor de los casos sería que se encontrase a DISP2 siendo atentido, y que tras ello DISP3 resultase estar preparado también y tuviese que ser atendido. Por tanto, su espera máxima sería menor que 2T.

          DISP2:  Este dispositivo, dado que sólo es atendido sin DISP1 no demanda servicio, está condicionado a lo que haga DISP1.  Por tanto, para este dispositivo no se puede garantizar nada. Si DISP1 siempre estuviera preparado, DISP2 nunca sería atendido.

          DISP3:  Este dispositivo, tras quedar preparado para recibir servicio, el peor de los casos que se podría encontrar sería que o bien DISP1 o bien DISP2 estuviese siendo atentido, pero nótese que o bien se atendrería a DISP1, o bien se atendería a DISP2, en una única iteración del bucle nunca se atendería a ambos. Por tanto, su espera máxima sería menor que T.

          Por tanto, el dispositivo al que se le puede garantizar un mejor tiempo de servicio sería a DISP3.

          Un programador Linux pretende garantizar la exclusión mutua entre dos procesos en el uso de un recurso compartido, usando para ello las siguientes llamadas al sistema POSIX: 

          • pause(): llamada al sistema que bloquea al proceso llamante
          • kill(int PID, int sig): llamada al sistema que envía una señal a un proceso cuyo identificador es PID. Si la señal es SIG_USR1, su efecto es que reanuda al proceso, si este está bloqueado en una llamada a pause(). Si no está bloqueado en dicha llamada, la señal no tiene efecto.

          Ambos procesos comparten una variable booleana ocupado, que es true si el recurso compartido está siendo usado, y false en caso contrario, siendo false su valor inicial, y ambos procesos utilizan los siguientes protocolos de entrada y salida a la región crítica:

          /* Protocolo de entrada a sección crítica */
          if(ocupado)
            pause();        // si encontramos recurso ocupado, esperamos

          ocupado= true;    // marcamos recurso como ocupado

           /*-----------------*/
           /* SECCIÓN CRÍTICA */
           /*-----------------*/

          /* Protocolo de salida de sección crítica */
          ocupado= false;             // marcamos recurso como libre
          kill(pid_compañero(), SIG_USR1);   // Despertamos al compañero(si está bloqueado)

          Se consigue garantizar de esta forma la exclusión mutua entre los dos procesos?
          Respuesta seleccionada:
          [No se ha dado ninguna]
          Respuestas:
          Comentarios para respuesta:

          No... Es imposible garantizar lo que se pretende de esta forma, pues no se garantiza la atomicidad entre la comprobación de la condición if (ocupado) y la acción que se lleve a cabo en base a ella (hacer pause() o seguir adelante). Supongamos por un momento que un el primer proceso realiza la comprobación if(ocupado) y resulta ser false (o incluso while(ocupado), el resultado sería el mismo). La decisión que se toma es no hacer el pause() y directamente ejecutar ocupado=true... pero justo antes de ejecutar esta instrucción es apropiado por el otro proceso que comprueba ocupado (aún vale false, el primer proceso no tuvo tiempo de ponerlo a true), lo pone a true, y entra en la sección crítica. Si estando ese proceso en la sección crítica el primer proceso se reactiva, pondría ocupado a true (de hecho, ya lo estaba) y entraría en la sección crítica, saltando por los aires la exclusión mutua.
